使用GSAP简单实现react列表中的下拉刷新和上拉加载更多

January 11, 2017

项目上线后看着列表不能下拉刷新不能上拉加载非常不爽，就做了个简单的实现。

为什么自己造轮子？用react不想添加jQuery，同时GitHub上的大部分代码都不能同时支持上拉和下拉。

准备

找个现成的项目新建一个页面，写个timeout模拟异步请求。

refresh() {
  setTimeout(()=>{this.setState({
    items: [1,2,3,4,5]
  })}, 1000);
}

loadMore() {
  setTimeout(()=>{this.setState({
    items: this.state.items.concat([1,2,3,4,5])
  })}, 1000);
}

再整个生成列表。

function renderList(list) {
  return (list.map((v, i) => (
    <li key={i} onClick={function() {browserHistory.push('/MerchantDetail/' + v.id + '/' + v.distance);}}>
      <div styleName="m-shop-right">
        <h1>
          {i}
        </h1>
      </div>
    </li>)
    )
  );
}

然后开始搞事情。

实现下拉和上拉

设计

现在的上拉和下拉大部分是：

  1. 监听ontouchstart, ontouchmove, ontouchend三个事件。
  2. 判断上拉下拉边界，进行对应的CSS transform和transition。
// sample，监听list的事件，修改list-header
<div id="list">
    <div id="list-header" />
    <div id="list-items" />
</div>

实现

最开始想用 react-css-transition-group 或 react-transition-group 来实现上拉和下拉效果，像这样：

<reactcsstransitiongroup ref={(el) => this._ref = el}>
    <div id="list-header" />
    <div id="list-items" />
</reactcsstransitiongroup>

然而仔细阅读了之后，发现只能在 mount 和 unmount 这两个关键点触发动画，在 update 的时候无法触发。

我非常不希望通过 <div style=/> 的方式，每次滑动的时候就render一次。于是去找GitHub提了issue，和大佬对话的传送门，大意就是说现在没法直接获得list-header的ref，得 findDOMNode(this._ref).children[0]。

这个就非常微妙了，既然如此，我为何一定要用 react-css-transition-group 或 react-transition-group 呢？简单点不好么？直接把这部分动画操作放在直接的DOM操作里绕开react，然后在ontouchend里回调，也是一种不错的思路。

<div id="g-wrap" onTouchStart={(e) => this.handleTouchStart(e)} 
  onTouchMove={(e) => this.handleTouchMove(e)} 
  onTouchEnd={(e) => this.handleTouchEnd(e)}> 
    // ...
</div>

handleTouchStart(e) {
  this._touchStartPosY = e.touches[0].pageY;//记录触摸开始位置
  this._gmainDefaultPaddingTopRem = '1.8rem';//记录初始列表头部位置，可改成接收参数
  this._gmainDefaultPaddingTop = parseInt(
    this._gmainDefaultPaddingTopRem.substr(0, this._gmainDefaultPaddingTopRem.length - 3));
  this._gmainDefaultPaddingBottom = 0;//底部
  this._lastScrollY = -1;//用于记录上一次滚动位置
  this._equalScroll = false;//用于标记上拉到了底部
}

handleTouchMove(e) {
  this._touchEndPosY = e.touches[0].pageY;

  // 下拉到顶
  if (window.scrollY <= 0) {
    this._lastScrollY = window.scrollY;
    // 拉且限制拉的距离
    if (this._touchEndPosY - this._touchStartPosY > 0 && this._touchEndPosY - this._touchStartPosY < 100) {
      let gmain = findDOMNode(this._gmain);
      // 偷懒使用TweenMax来生成动画
      TweenMax.to(gmain, 0, 
        {paddingTop: this._gmainDefaultPaddingTop + 
            (this._touchEndPosY - this._touchStartPosY)/20 + 'rem', 
            ease: Power0.easeInOut});
    }
  } else {
    // 上拉到底
    if (window.scrollY == this._lastScrollY) {
      if (!this._equalScroll) {
        this._equalScroll = true;
        this._equalScrollY = window.scrollY;
        this._equalClientY = e.touches[0].clientY;
      } 
      let gmain = findDOMNode(this._gmain);
      e.preventDefault();
      // 限制上拉距离生成动画
      let offsetY = this._touchEndPosY>100?100:this._touchEndPosY;
      TweenMax.to(gmain, 0, 
        {paddingBottom: this._gmainDefaultPaddingBottom + offsetY/20> + 'rem', 
        ease: Power0.easeInOut});
      
    } else {
      this._lastScrollY = window.scrollY;
    }
  }
}

handleTouchEnd(e) {
  if (window.scrollY <= 0) {
    if (this._touchEndPosY - this._touchStartPosY > 0) {
      let gmain = findDOMNode(this._gmain);
      // 弹回
      TweenMax.to(gmain, 1, 
        {paddingTop: this._gmainDefaultPaddingTopRem, 
            ease: Back.easeInOut});
      this.refresh();
    }
  } else {
    if (this._equalScroll) {
      let gmain = findDOMNode(this._gmain);
      // 弹回
      TweenMax.to(gmain, 1, 
        {paddingBottom: this._gmainDefaultPaddingBottom, 
            ease: Power3.easeInOut});
      this.loadMore();
    }
  }
}

总结

整个思路就是：通过起始 pageY 和结束 pageY 判断用户垂直滑动方向，在根据 window.scrollY 和保存的上一次滑动位置来判断是否到顶（或者到底），到顶（或者到底）后执行 tansform，touchEnd的时候进行回调。

当然这只是个简单实践，还需要进一步优化，比如：

  • 支持传入回调函数。
  • 支持定义限制的上拉下拉高度。
  • 支持上拉下拉时载入自定义的静态资源，如活动动画或是刷新效果等。
  • 设置默认参数。
  • 将TweenMax改为原生方法实现减少大小。
  • 抽取为high-order-component。
  • 最近这种纯前端的东西做的有点多，是时候拿 nodejs 和 express/koa 搞事情了。
