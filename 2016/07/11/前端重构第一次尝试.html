前端重构第一次尝试

July 11, 2016

前段时间自行吐槽了自己参与大部分工作的前端项目，慢慢学习前端工作流到现在，并尝试了重构，记录一下。

配置管理

首先，把和环境相关的配置提取出来，使用webpack.DefinePlugin，避免每次部署时开发环境、测试环境、生产环境的变量手动修改。

/* 定义运行环境相关变量 */
var SITE_STATIC_PREFIX = '/yh-wifi-web/';
/* ... */

module.exports = {
  plugins: [
    new webpack.DefinePlugin({
      SITE_STATIC_PREFIX: JSON.stringify(SITE_STATIC_PREFIX),
      SITE_DYNAMIC_PREFIX:JSON.stringify(SITE_DYNAMIC_PREFIX)
    })
  ]
}；

资源管理

然后，把css和图片等资源文件从在index.html里直接饮用挪到js中引用，以便将静态资源文件纳入管理，解决热部署问题（大公司里怎样开发和部署前端代码？ - 回答作者: 张云龙）。

/* 引用静态资源 */
import zoneMomImage from '../../../image/mom.png';
import styles from '../../../css/app.css';

在webpack配置中添加对应loader。

/* 在webpack配置中添加对应loader */
module.exports = {
  module: {
    loaders: [{
      test: /\.css$/,
      loader: loaders: ['style?sourceMap', 'css?modules&camelCase&sourceMap'],
      include: path.join(__dirname, 'css')
    }, {
      test: /\.(png|jpg|gif|svg)$/,
      loaders: ['url?limit=8192'],
      include: path.join(__dirname, 'image')
    }]
  }
}；

这样，就可以在react中使用css module。

.sample-camel{
  color: red;
}

在react中引用之后，可以使用对应变量表示样式

render(
  return <div className={styles.sampleCamel} />
)

然而，考虑到现在的前端开发流程，UE->UI->HTML+CSS->react组件，以及现有项目的修改难度，引入了react-css-modules。

// ComponentSample.js
import CSSModules from 'react-css-modules';
// 修改className为styleName
render(
  return <div styleName="sample-camel" />
)
// 装饰组件
export default CSSModules(ComponentSample, styles, {allowMultiple:true});
// webpack配置
module.exports = {
  module: {
    loaders: [{
      test: /\.css$/,
      loader: loaders: ['style?sourceMap', 'css?modules&importLoaders=1&localIdentName=[path]__[name]__[local]__[hash:base64:5]'],
      include: path.join(__dirname, 'css')
    }, {
      test: /\.(png|jpg|gif|svg)$/,
      loaders: ['url?limit=8192'],
      include: path.join(__dirname, 'image')
    }]
  }
}；

这样，不论是现有项目修改还是新项目从静态页面到react，都更为方便。

最后修改在生产环境的webpack配置。

module.exports = {
  output: {
    path: path.join(__dirname, 'asset'),
    filename: '[name].[chunkhash].js',
    publicPath: '/'
  },
  resolve: {
    extensions: ['', '.js']
  },
  devtool: false,
  plugins: [
    new webpack.optimize.OccurenceOrderPlugin(),
    new webpack.DefinePlugin({
      'process.env': {
        'NODE_ENV': JSON.stringify('production')
      }
    }),
    new webpack.optimize.DedupePlugin(),
    new webpack.optimize.UglifyJsPlugin({
      compress: {
        warnings: false
      }
    }),
    new webpack.DefinePlugin({
      SITE_STATIC_PREFIX: JSON.stringify(SITE_STATIC_PREFIX),
      SITE_DYNAMIC_PREFIX:JSON.stringify(SITE_DYNAMIC_PREFIX)
    }),
    new ExtractTextPlugin('[name].[chunkhash].css', {allChunks:true}),
    new HtmlWebpackPlugin({
      template:'scripts/index.ejs',
      minify:{
        removeComments: true,
        collapseWhitespace: true,
        removeRedundantAttributes: true,
        useShortDoctype: true,
        removeEmptyAttributes: true,
        removeStyleLinkTypeAttributes: true,
        keepClosingSlash: true,
        minifyJS: true,
        minifyCSS: true,
        minifyURL: true
      },
      inject:true
    })
  ],
  module: {
    loaders: [{test: /.jsx?$/,loaders: ['babel'],include: path.join(__dirname, 'scripts')}, {test: /.js?$/,loaders: ['babel'],include: path.join(__dirname, 'scripts')}, {test: /.css$/,loader: ExtractTextPlugin.extract('style', 'css?modules&importLoaders=1&localIdentName=[path]__[name]__[local]__[hash:base64:5]'),include: path.join(__dirname, 'css')}, {test: /.(png|jpg|gif|svg)$/,loaders: ['url?limit=8192', 'image-webpack?bypassOnDebug&optimizationLevel=7&interlaced=false'],include: path.join(__dirname, 'image')}]
  }
};

其中尝试加入图片压缩，但在windows环境下imagemin依赖的库会出现问题，目前没有解决方案，只能考虑在其他步骤中进行处理。

代码规范

按照以下进行了代码重写。 1. 重申命名规范，组件/页面单独文件夹，文件夹下index.js作为默认导出，使用到的多个组件在文件夹内创建[ComponentName.js]来引用。 2. 按照要求将react-router的hashHistory替换为browserHistory，在上面的修改完成后，由于使用browserHistory导致的资源路径问题也同步解决了。
